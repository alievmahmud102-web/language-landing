# SEO rules

## Landing priorities

- **Speed** and **mobile UX** directly affect conversion; treat SEO and CRO as one system.
- Unique `<title>` and meta description per important page; avoid duplicate thin content.

## Technical

- Clean URLs, stable slugs; use `href` for navigation that should be indexed (not only client-side-only routes without fallbacks if required).
- Proper heading hierarchy (`h1` once per view where it makes sense).
- `robots.txt` and sitemap aligned with what should be indexed; no accidental indexing of admin or staging.

## Content

- Clear H1 and opening paragraph aligned with search intent (local language courses, exam prep, IT courses—match actual center positioning).
- Structured data (e.g. `LocalBusiness` / `Course`) when accurate and maintained.

## Measurability

- Events or UTM discipline for **source attribution** (feeds dashboard); document parameter conventions in `docs/`.
