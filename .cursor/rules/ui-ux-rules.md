# UI/UX rules

## Audience

- **Operators** at education centers (small/medium teams): need clarity, density where useful, and low training cost.
- **Parents / leads** on the landing: scan quickly, trust signals, one primary CTA path.

## Language

- **Russian** for all user-visible strings in the product UI and marketing surfaces unless a specific English-only screen is agreed.

## Principles

- One primary action per screen or section; secondary actions visually de-emphasized.
- Consistent spacing, type scale, and components across admin, CRM, and dashboard.
- **Feedback**: loading, success, and error states on every async action (forms, saves, filters).

## Forms

- Labels above fields; inline validation after blur or submit; errors in plain language.
- Preserve user input on validation failure.

## Accessibility

- Keyboard focus order, visible focus, sufficient contrast; semantic headings on long pages.

## Mobile

- Critical flows (view lead, update status, submit inquiry) must work well on **phone** widths.
