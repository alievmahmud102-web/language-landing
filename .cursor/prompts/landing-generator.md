# Landing generator — prompt template

Use this when generating or rewriting a **conversion-focused landing** for an education center (language schools, IT courses, exam prep). Output: **solution → explanation → next steps**. Keep it concise; include short examples where useful.

## Inputs (fill or ask)

- Center name, city, primary offer (e.g. “IELTS group from March”).
- Target parent/student segment and age range of students.
- Proof: teachers, results, reviews, numbers (students, years, pass rate—only if true).
- Primary CTA: callback, trial lesson, diagnostic, Telegram—one main path.
- Secondary: phone, messengers, address.
- Tone: formal / friendly; taboos (claims you cannot make).

## Required sections in the deliverable

1. **Above the fold**: headline, subhead, primary CTA, trust strip (3 bullets max).
2. **Pain → outcome**: 3 pains, 3 outcomes (short).
3. **Programs / formats**: cards with 1-line benefit each.
4. **How it works**: 3–4 steps.
5. **Social proof**: quotes or logos (placeholders marked if missing).
6. **FAQ**: 5–7 objections (price, schedule, level, trial, refunds—only what applies).
7. **Final CTA block** + microcopy for the form (labels in **Russian** if this ships to RU users).

## Technical hints for implementation

- Semantic sections, one clear `h1`, meta title/description suggestions.
- Mobile-first section order; avoid hero images that hurt LCP without a plan.

## Next steps

- List 3 A/B hypotheses (headline, CTA label, form length) and what metric each would move.
