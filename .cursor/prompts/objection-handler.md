# Objection handler — prompt template

Use to respond to **sales objections** for the education-center system. Format: **solution → explanation → next steps**.

## Common objections (seed list)

- “We already have a website on Tilda / WordPress.”
- “We use AmoCRM / spreadsheets.”
- “Too expensive vs a freelancer.”
- “We don’t have time to migrate.”
- “We need custom feature X.”

## For each objection, output

1. **Acknowledge** (one sentence, no arguing).
2. **Clarify** (one question to find the real blocker).
3. **Reframe** around **system outcome** (speed to lead, source visibility, fewer lost inquiries).
4. **Proof path** (case, demo, pilot milestone).
5. **Micro-commit** (15-min call, send 3 screenshots, trial checklist).

## Positioning anchor

- Competitors sell **assets** (site, template, isolated CRM). You sell a **pipeline**: attract → capture → follow up → measure.
